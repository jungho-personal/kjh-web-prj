## 🧩 작업 내용

### 1. Top Navigation 제거 및 레이아웃 단순화
- 기존 TopNav + Sidebar 구조에서 TopNav를 제거
- 전체 레이아웃을 Sidebar + Content 구조로 통합
- 화면 분할 및 시선 흐름을 단순화하여 구조 명확화

### 2. AppShell 구조 정리
- Sidebar는 고정, Content 영역은 `flex-1`로 설정
- 페이지별 레이아웃 책임을 AppShell이 아닌 Content로 위임
- 스크롤 영역 분리 및 안정적인 SPA 구조 확립

### 3. Content 폭 / 여백 문제 원인 분석 및 해결
- 오른쪽 여백이 줄어들지 않는 문제 발생
- 원인이 Sidebar나 AppShell이 아닌,
  Content 내부의 `max-width + padding` 조합임을 확인
- Content 레이아웃 기준을 재정의하여
  Sidebar 바로 옆에서 자연스럽게 콘텐츠가 시작되도록 수정

### 4. 페이지 헤더 위치 및 스타일 통일
- Blog / Dev Log 기준으로
  Portfolio / Resume / Playground 페이지 헤더 구조 통일
- 모든 페이지에서 시작 위치 및 타이포그래피 일관성 확보

### 5. Sidebar UI 디테일 개선
- Sidebar 프로필 영역 정렬 개선
- Navigation 그룹 시각적 정리
- 전체적인 여백과 밀도 조정으로 가독성 향상
